make some sort of a table or website page where I can browse my chats and look at paragraph capsule summaries plus a hyperlinked smattering of the key figures or topics which can branch out into other database play

You want a “Chats Index” page that behaves like a bibliographic catalog plus a jump-pad into entities/tables/artifacts. The right unit is: one conversation = one row/card, with (a) a paragraph capsule summary, (b) a small set of hyperlinked entity/topic chips, and (c) quick pivots into other views.

Here’s a concrete design you can implement in EsoPipe with minimal new concepts.

A. The page you should build: Chats Browser

Main layout (two-pane)
Left: filters + search
Right: results list (cards or table), with pagination/infinite scroll.

Filters (facets)

Date range (start/end)

Era focus (derived: ancient/medieval/renaissance/modern psych/etc.)

Intent tags (once you have them)

“Has tables” / “Has artifacts”

Top entity (dropdown from entities.json)

Confidence/coverage (optional later)

Search

Full-text keyword search (FTS5 on conversation title + snippets)

“Smart search” (later: embeddings)

Result item (the “capsule”)
Each conversation shows:

Date + message count

“Key entities/topics” chips: 6–12 clickable items, mixed types (thinkers/texts/concepts/tools)

Quick actions:

Open full transcript

View artifacts derived from this conversation (count)

C. How to produce capsule summaries (deterministic + LLM, but controlled)

You have 249 conversations now, and more later. You want summaries that are:

short

consistent

not hallucinated

tied to the actual conversation

Recommended approach:

Deterministic extract
For each conversation, derive:

top N entities (by mention count / tf-idf / centrality)

top N “turn snippets” (e.g., the 3 user turns with most entities + 2 assistant turns with tables)

any extracted table titles/templates present

LLM summarizer (bounded)
Prompt an LLM with:

the extracted snippets (not the full conversation)

the top entities list

explicit constraints: “One paragraph. Mention 3–6 entities from the list. Do not invent citations. If uncertain, hedge.”

Validation pass (deterministic)

length bounds (e.g., 60–120 words)

must include at least 3 entity labels from the top list

must not include entities not present (simple string/alias check)
If it fails, regenerate once; if still fails, fall back to a deterministic summary (“Conversation about X/Y/Z; includes tables; main themes …”)

This is exactly “specification engineering” applied to summaries.

D. How the “hyperlinked smattering” should work

Entity chips should link to entity pages you already support:

/entities/:id

On each entity page, add “Seen In Conversations” tab:

list of conversations (sorted by weight or recency)

show capsule excerpt + “jump to snippet”

That gives you branching exploration:
Chat → Entity → Other chats → Tables → Artifacts.

It becomes the UI surface for intent tagging later (“this conversation is 60% /compare, 40% /audit”)

It’s the natural place to add a “Generate artifact from this conversation” button later
